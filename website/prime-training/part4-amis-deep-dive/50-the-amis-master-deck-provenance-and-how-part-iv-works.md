# The AMIS Master Deck: Provenance and How [Part IV](./50-the-amis-master-deck-provenance-and-how-part-iv-works.md) Works


## What This Deck Is

[Part IV](./50-the-amis-master-deck-provenance-and-how-part-iv-works.md) integrates the Prime AMIS Training Master Deck (draft, 63 slides) — the AMIS-specific training referenced during Day 2's materials discussion ([Section 31.3](../part2-hands-on/31-lab-environment-options-for-consultants.md): 'the Ryan AMIS training… it's very specific'). Its closing slide credits Sean Doyle, Justin Mitchell, and Nada Khaled for source material, and its copyright block dates the core content to the 2018 Dell/RSA era — making this the oldest artifact in the program and, in effect, its ancestor: comparison shows Nadine's V8 master ([Part III](../part3-deck-companion/42-the-training-deck-overview-and-how-to-use-part-iii.md)) inherited entire modules from this deck verbatim, including the network/port matrix and the sixteen-slide API cookbook. Where [Part III](../part3-deck-companion/42-the-training-deck-overview-and-how-to-use-part-iii.md)'s deck is the current curriculum, this deck is the specialist's sourcebook — deeper on AMIS internals, developer integration tooling, and troubleshooting than anything else in the program.

## The Hidden-Slide Discovery

A structural finding worth recording: eleven of the deck's 63 slides are marked hidden in the file — slides 25–31 (the manual installation and configuration module: rsautil, keytool, Template Builder, the AM-side configuration checklist, and the network matrix) and slides 58–61 (the legacy service-management, upgrade, hardening, and first troubleshooting slides). The pattern is unmistakable: the hidden slides are precisely the material Prime Quick Setup and PrimeKit obsoleted — independent confirmation of the Day 1 history ([Section 6.4](../part1-foundations/6-installation-walkthrough-tarball-scripts-and-quick-setup.md): 'much legacy training documentation predates this automation'). [Part IV](./50-the-amis-master-deck-provenance-and-how-part-iv-works.md) restores and annotates them anyway, for two reasons: legacy customers still run pre-PrimeKit layouts these slides describe, and several hidden slides (notably 27's AM-side checklist and 60's hardening inventory) document things no current material states explicitly. Hidden slides are flagged as such in their commentary.

## Integration Policy for [Part IV](./50-the-amis-master-deck-provenance-and-how-part-iv-works.md)

Because this deck overlaps its descendants, [Part IV](./50-the-amis-master-deck-provenance-and-how-part-iv-works.md) applies a stricter economy than [Part III](../part3-deck-companion/42-the-training-deck-overview-and-how-to-use-part-iii.md): slides carrying new or deeper material get full annotation; slides that are verbatim duplicates of [Part III](../part3-deck-companion/42-the-training-deck-overview-and-how-to-use-part-iii.md) content — chiefly the API cookbook (slides 40–55, identical to [Part III](../part3-deck-companion/42-the-training-deck-overview-and-how-to-use-part-iii.md) slides 70–85) and the network matrix (slide 31 = [Part III](../part3-deck-companion/42-the-training-deck-overview-and-how-to-use-part-iii.md) slide 10) — are cross-referenced in a consolidated note rather than re-embedded, so the guide never teaches the same slide twice. [Chapter 56](./56-amis-deck-reconciliation-map.md) reconciles all 63 slides. Legacy path note: this deck predates PrimeKit's standard layout, so its examples use paths like /local/apps/… and C:\\RSA\\… alongside the modern /opt/rsa/primekit — both forms are preserved in commentary because consultants meet both in the field.
